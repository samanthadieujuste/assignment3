The main difference is that in the Integration Manager workflow, developers push changes to their own public repository and an integration manager pulls those changes and meges them into the central repository. While in a Dictator and Lieutenants workflow developers push changes to lieutenant repositories and lieuenants mege and manage these changes before the dictator merges the lieutenant repositories into the central repositoy. 

So basically, the lieutenants act as another layer of "checks" to a developers work.

10.
With a larger team there are a lot of changes that could occur. A Dictator and Lieutenants workflow distributes the work of reviewing and merging code which reduces conflict and improves managability.
